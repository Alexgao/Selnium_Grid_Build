Selenium Grid Build.

这个页面主要介绍如果搭建Selenium Grid环境:

Selnium Grid Wiki：

首先需要下载一个

Selenium Server 的jar包.

配置hub主机

配置hub和node分两种方法配置1： cmd直接跟上参数，2：加上json文件

java -jar selenium-server-standalone-xx.xx.xx.jar -role hub -port 4444

配置node节点

java -jar selenium-server-standalone-2.14.0.jar -role node -port 5555  -hub http://localhost:4444/grid/register -browser browserName=firefox,version=3.6,maxInstances=5,platform=LINUX,firefox_binary=xxxxx -Dwebdriver.ie.driver="D:\Drivers\IEDriverServer.exe"

注意：

Browser： { browserName需要写正确的名字，以便查看 firefox, chrome, internet explorer version版本随意写（尽量写正确的版本，以便管理），并不和启动的浏览器相关联。maxInstances为启动的最大线程数（即浏览器允许启动的个数）， firefox_binary为firefox.exe的路径（具体到firefox.exe）。 } Hub: 指定hub主机地址：必须这么写 -hub http://localhost:4444/grid/register Port： 为指定的端口, 默认的是5555，但是在配置的时候最好自己给定，以免几台电脑都是用默认从而冲突。 Dwebdriver.ie.driver： 指定ie driver在node机子上的位置chrome为 Dwebdriver.chrome.driver

使用json配置：

java -jar selenium-server-standalone-xx.xx.xx.jar -role hub –hubConfig hubConfig.json
java -jar selenium-server-standalone-xx.xx.xx.jar -role node –nodeConfig nodeConfig.json

配置好hub和node的时候基本上就可以查看是否配置成功： http://localhost:4444/grid/console

多台电脑配置只需要修改localhost为hub的ip地址就好了 Node config json

常见错误

主机地址或者端口写错：

浏览器地址如果有空格类似C:\Program Files\Mozilla Firefox 那么 -browser “browserName=firefox,version=3.6,maxInstances=5,platform=LINUX,firefox_binary=C:\Program Files\Mozilla Firefox” -browser 对应的参数值必须用”双引号”括起来。

Selenium Grid对我来说一般使用时分布式测试，那么怎么才能体现出分布式测试呢， 只要在我们的测试代码中写出多线程操作grid就是自动给我们分配到node电脑上去操作。
